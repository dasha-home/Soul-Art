# Даша Художник — арт-платформа (SPA)

Одностраничное приложение на чистом HTML, CSS и Vanilla JavaScript. Интерактивный портал в мир рисунков Даши.

---

## Где находится загрузчик (админка)

Загрузчик фото — это **отдельная комната на сайте**: форма, где Даша выбирает файл с компьютера и загружает его на сайт.

**Одна ссылка, по которой всё открывается (сохраните в закладки):**

**https://dasha-home.github.io/Soul-Art/admin.html**

По этой ссылке откроется пригласительная страница, внизу будет надпись **«Для Даши: загрузчик фото (админка)»** — нажмите её, откроется комната с формой загрузки (поля: файл, название, подпись, кнопка «Загрузить на сайт»). Либо нажмите «Войти» и затем в верхнем меню — **«Загрузить фото»**, или внизу страницы — **«Ссылка на админку»**.

**Если заходите с самого сайта:** после нажатия «Войти» в шапке появятся пункты «Галерея», «О Даше», **«Загрузить фото»** — это и есть админка. Внизу страницы тоже есть ссылка «Ссылка на админку (загрузчик фото)».

## Архитектура

- **`index.html`** — точка входа, инициализация слоёв интро и контейнера SPA.
- **`styles.css`** — оформление, анимации, CSS‑переменные (`--main-speed`, `--accent-color`).
- **`main.js`** — управление состояниями (`INTRO`, `GALLERY`, `ABOUT`), интро‑сцена и галерея.

Основной контейнер SPA — элемент `main#app-content`. Всё содержимое режимов (галерея, о художнике и будущие разделы) рендерится в него динамически в зависимости от состояния `AppState`.

## Состояния

- `INTRO` — стартовая сцена: слой Фудзи, туман, заголовок «Рисунки Даши» и персонаж Даша.
- `GALLERY` — галерея слайдером. Данные из `data/artworks.json` (редактирование через Prose.io или админку на сайте).
- `ABOUT` — статический текст о художнике (можно свободно менять и расширять).
- `ADMIN` — загрузка фото с компьютера на сайт (кнопка «Загрузить фото» в шапке).

## Загрузка фото прямо с сайта (админка)

Загрузчик находится на том же сайте: кнопка **«Загрузить фото»** в верхнем меню или прямая ссылка **адрес_сайта/#admin** (см. раздел «Где находится загрузчик» выше).

Даша может:

1. Нажать **«Загрузить фото»**.
2. Один раз ввести **токен GitHub** (см. ниже) и нажать «Сохранить и открыть загрузчик».
3. Выбрать файл (картинку), ввести название и подпись, нажать **«Загрузить на сайт»**.

Фото попадёт в репозиторий в папку `data/images/`, а запись о работе — в `data/artworks.json`. Галерея на сайте обновится после обновления страницы.

**Как получить токен:** зайти на GitHub → **Settings** → **Developer settings** → **Personal access tokens** → создать токен с правом **repo**. Скопировать токен и вставить в админке. Токен хранится только в этой вкладке браузера до закрытия.

## Как Даше добавлять работы через Prose.io

[Prose](https://prose.io) — редактор контента для GitHub. Даша может заходить в репозиторий и править галерею без командной строки.

1. Открыть в браузере: **https://prose.io/#dasha-home/Soul-Art**
2. Войти через GitHub (если ещё не вошли).
3. В списке файлов открыть папку **data** и файл **artworks.json**.
4. В `artworks.json` — массив `artworks`. Каждая работа — объект с полями:
   - **id** — уникальный короткий ключ (латиница, например `sunset`).
   - **title** — название работы.
   - **subtitle** — подпись (серия, год, техника — по желанию).
   - **imageUrl** — путь к картинке: `./01_fuji.png` для файлов в корне или `./data/images/имя.png` для картинок в папке `data/images/`.
5. Чтобы добавить новую картинку: в Prose можно создать папку **data/images**, загрузить туда файл (кнопка загрузки), затем в **artworks.json** добавить новый элемент с `"imageUrl": "./data/images/имя_файла.png"`.
6. Сохранить (Commit) — изменения попадут в репозиторий, сайт подхватит их при следующей загрузке.

В корне репозитория есть файл **`_prose.yml`**: в нём указана папка `data/images` для загрузки изображений. В Prose при редактировании markdown-файлов можно вставлять картинки — они будут загружаться в эту папку. Для галереи сайта всё равно нужно добавлять запись в `data/artworks.json` (вручную или через админку на сайте).

Ссылка «Добавить или изменить работы в галерее (Prose)» в блоке галереи ведёт в этот же репозиторий в Prose.

## Данные и GitHub

Галерея загружает список работ из `data/artworks.json` (сначала с raw.githubusercontent.com, при ошибке — локально). Формат: `{ "artworks": [ { "id", "title", "subtitle", "imageUrl" }, ... ] }`.

## Медиафайлы

Ожидаемые файлы (можно положить в папку `assets/` рядом с HTML):

- `01_fuji.png` — задний план.
- `00_start_clouds.png` — туман/облака.
- `02_dasha.png` — персонаж Даша.
- Любые placeholders для галереи (или реальные рисунки) в полях `imageUrl`.

## Добавление новых «комнат»

1. Добавьте новое значение в `APP_STATES` в `main.js`.
2. Создайте функцию рендера (по аналогии с `renderGallery` или `renderAbout`).
3. Добавьте кнопку навигации в шапку (`.top-nav__link` с `data-state="NEW_STATE"`).
4. Внутри `renderState` обработайте новое состояние.

Такой подход не требует переписывать существующую логику и позволяет расширять портал постепенно.
